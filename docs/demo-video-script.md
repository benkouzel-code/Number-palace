# Number Palace Demo Video Script

Target length: about 2 minutes 35 seconds. Record at 1080p, keep the browser
zoom near 100%, turn on the pointer highlight if available, and speak while the
matching feature is visible.

Before recording:

- Start with a clean or demo-ready browser profile.
- Verify the deployed link is accessible without an owner-only sign-in.
- Prepare one short custom sequence, such as `201314`, named "Contest route."
- Keep the GitHub README open in a second tab for the development segment.
- Verify the Codex model label before saying "GPT-5.6."

## 0:00-0:15 - Hook

**On screen:** Landing page, then hover over **Enter the quick demo**.

**Voiceover:**

> Long numbers are difficult because they are abstract. Number Palace turns
> every digit into part of a vivid scene, then trains you to decode that scene
> back into the exact sequence.

## 0:15-0:35 - Explain the system

**On screen:** Enter the quick demo and open the master codex.

**Voiceover:**

> Each player creates a personal master list. Every digit from zero through
> nine gets a noun, an adjective, and a verb. You can customize every field, or
> start with one of three example sets like Hercules, burning, and lifting.

## 0:35-0:58 - Generate a palace

**On screen:** Return home, select six digits, generate a random route, and
slowly show the scene cards.

**Voiceover:**

> I can generate a random number hash at any length from three to thirty-six
> digits. Multiples of three work especially well because playback always
> follows noun, adjective, verb. Each group becomes a strange mental room that
> is easier to remember than raw digits.

## 0:58-1:15 - Save a useful sequence

**On screen:** Enter `201314`, name it "Contest route," and save it.

**Voiceover:**

> I can also enter an exact sequence, give it a unique name, and save it. That
> makes the game useful for repeatedly practicing a specific combination
> instead of always using random numbers.

Do not use a real PIN, account number, or other secret in the recording.

## 1:15-1:42 - Number-by-number practice

**On screen:** Start number-by-number mode. Enter one correct digit, then
demonstrate the three hint buttons on the next digit.

**Voiceover:**

> Number-by-number mode isolates each recall step. If I get stuck, the first
> hint tells me the role, the second shows that digit's complete codex entry,
> and the third reveals the number. That progression gives me help without
> throwing away the whole retrieval attempt.

## 1:42-1:58 - Whole-sequence practice

**On screen:** Switch to whole-sequence mode. Show the single answer field and
the attempt counter. You can make one intentional miss, but do not spend time
using all three attempts.

**Voiceover:**

> Whole-sequence mode tests the entire route in one shot. It tracks each try
> and reveals the answer after three misses, so the learner can study the gap
> and try again.

## 1:58-2:25 - Codex and GPT-5.6

**On screen:** Briefly show the GitHub README, project files, test command, and
the original environment image. If useful, use a pre-recorded two-second clip
of the Codex task rather than exposing private conversation details.

**Voiceover:**

> I built Number Palace in Codex with GPT-5.6. Codex translated the product
> brief into the typed React game flow, helped generate and integrate the
> original palace artwork, implemented both practice modes, and ran the build
> and smoke test. It also accelerated iteration: when I changed the memory
> grammar, Codex traced the feature and made configuration, playback, and
> recall consistently use noun, adjective, verb. I made the key learning
> decisions, while Codex made it practical to implement and verify them as a
> cohesive product.

If the core task used more than one model, replace the first sentence with an
exact description of which work was done in GPT-5.6.

## 2:25-2:35 - Close

**On screen:** Return to the palace home screen with a completed route and
stats visible.

**Voiceover:**

> Number Palace makes exact recall visual, personal, and fun. The live app and
> complete source are linked in the submission.

## Editing checklist

- Keep the final export under 3:00.
- Use a public YouTube visibility setting, not unlisted if the rules say public.
- Include spoken audio for both Codex and GPT-5.6.
- Show the working product for most of the runtime.
- Remove notifications, bookmarks, personal data, and API keys from the frame.
- Add captions and verify that "Codex," "GPT-5.6," and "Number Palace" are
  transcribed correctly.
